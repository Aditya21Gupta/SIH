Code: NOT_FOUND

ID: bom1::bzrnh-1757788875338-d47716e3df31

This error typically occurs when Vercel cannot find the deployment or the build output.

Common Causes

  • Incorrect vercel.json configuration
  • Wrong build output directory specified
  • Missing routes or rewrites in configuration
  • Project structure doesn't match Vercel expectations
  • Build process failed to generate expected output
  • Framework configuration issues
  • Missing required files for deployment

Solutions

  • Verify your vercel.json configuration
  • Check build output directory in project settings
  • Add proper routes and rewrites for your application
  • Ensure all required files are included in deployment
  • Test build process locally before deploying
  • Check framework-specific documentation for Vercel
  • Redeploy with correct configuration

Step-by-Step Troubleshooter

1. Check vercel.json Configuration

Ensure your vercel.json file is correctly configured for your framework.

// Example vercel.json for React.js
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "build"
      }
    }
  ],
  "routes": [
    {
      "handle": "filesystem"
    },
    {
      "src": "/(.*)",
      "dest": "/index.html"
    }
  ]
}

2. Verify Build Output Directory

Make sure your build output directory matches what's specified in your Vercel project settings.

# Common build commands
npm run build # Creates build/ directory for React
npm run export # Creates out/ directory for Next.js
yarn build # Alternative using Yarn

3. Check Deployment Logs

Examine your deployment logs in the Vercel dashboard for any build errors or warnings.

# You can check build logs locally with:
vercel logs [deployment-url]

# Or inspect previous deployments:
vercel list
vercel inspect [deployment-id]
